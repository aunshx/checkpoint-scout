So let's go ahead

We're building a frontend dashboard for Checkpoint Scout, the agent in this repo. The dashboard is a read-only review UI for Entire's GTM team to see the agent's findings.

Architecture decisions (locked, do not change):
- React + Vite + TypeScript + Tailwind
- Single page, no router
- Static build, no backend
- Deployed to GitHub Pages
- Data is a committed JSON file generated by a Python helper from the agent's outputs

Project structure:
- All web code in /web subdirectory
- Data file at /web/public/data.json
- Python data-builder at /scripts/build_dashboard_data.py
- GitHub Actions workflows at /.github/workflows/

Step 1: Create the React app

cd to project root, then:
- mkdir -p web && cd web
- npm create vite@latest . -- --template react-ts
- Press 'y' to overwrite if prompted
- npm install
- npm install -D tailwindcss postcss autoprefixer
- npx tailwindcss init -p
- npm install react-markdown remark-gfm lucide-react

Configure tailwind.config.js to scan ./index.html and ./src/**/*.{js,ts,jsx,tsx}
Replace src/index.css with:
@tailwind base;
@tailwind components;
@tailwind utilities;

Update vite.config.ts to add base: '/checkpoint-scout/' for GitHub Pages

Step 2: Build the Python data builder

Create /scripts/build_dashboard_data.py that:
- Finds the latest run in outputs/runs/
- Reads qualified.json
- For each pain moment, reads source.json, classification.json, and all three markdown artifacts (public_comment.md, outreach.md, case_study.md)
- Reads dashboard.md or computes the funnel numbers from the qualified set
- Writes a single web/public/data.json with this structure:

{
  "run_timestamp": "2026-05-11T14:50:17Z",
  "funnel": {
    "raw_candidates": 2774,
    "classified": 100,
    "qualified": 5,
    "artifacts_generated": 15
  },
  "pain_moments": [
    {
      "id": "e43473edca79af21",
      "rank": 1,
      "repo": "huggingface/transformers",
      "pain_type": "review_burden",
      "score": 8,
      "signal": "HF/transformers added a blanket 'Code Agent Policy' citing being overwhelmed and bottlenecked by agent-written PRs",
      "url": "https://github.com/huggingface/transformers/pull/...",
      "artifacts": {
        "comment": "<full markdown content as string>",
        "outreach": "<full markdown content as string>",
        "case_study": "<full markdown content as string>"
      },
      "classification": {
        "rationale": "...",
        "entire_relevance": "..."
      }
    }
  ]
}

Run the builder. Verify web/public/data.json is created.

Step 3: Build the dashboard

In /web/src/, build these components:

types.ts — TypeScript types for the data structure above

App.tsx — Top-level component. Fetches '/checkpoint-scout/data.json' on mount, stores in state, passes to children. Includes loading and error states.

Hero.tsx
- Title "Checkpoint Scout"
- Subtitle "AI-coding pain moments in public OSS, surfaced for Entire's GTM team"
- A short 2-line description: "Scans public GitHub for moments where open-source maintainers struggle with AI-generated code. Drafts useful artifacts for review."
- The Funnel component below

Funnel.tsx
- Four stat blocks in a row showing raw_candidates → classified → qualified → artifacts_generated
- Labels: "Raw candidates", "Classified", "Qualified pain moments", "Artifacts generated"
- Arrows between them (lucide-react ArrowRight)
- Below: a small caption "Last run: <run_timestamp> · Next scheduled run: daily 6am UTC (cron disabled until production credentials wired)"

PainMomentCard.tsx
- Shows: rank badge, score badge, repo name (monospace), pain_type chip, signal text
- Pain type chip color coding: review_burden=red, intent_unclear=amber, ai_authorship_concern=purple, other=slate
- Click toggles expanded state showing the ArtifactTabs component

ArtifactTabs.tsx

- Active tab renders the corresponding markdown using react-markdown with remark-gfm
- Style markdown nicely: headings, code blocks, links, lists all readable in dark mode
- Above the tabs: the source GitHub URL with external link icon and "Source PR" label

Footer.tsx
